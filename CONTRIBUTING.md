# Contributing to MaintainerOS

Thanks for your interest in contributing! MaintainerOS is built by and for
open-source maintainers, and contributions of every size are welcome.

## Ground rules

- Be kind. Follow the [Code of Conduct](./CODE_OF_CONDUCT.md).
- Open an issue before starting a large change so we can discuss the approach.
- Keep PRs focused — one logical change per PR.
- Add or update tests and docs when behavior changes.

## Getting started

```bash
bun install
bun run dev
```

## Pull request checklist

- [ ] The change has a clear purpose described in the PR body
- [ ] Tests pass locally
- [ ] Docs / README updated if relevant
- [ ] No new ESLint / TypeScript warnings
- [ ] No fake data, fake metrics, or deceptive copy

## Reporting bugs

Use the bug report issue template and include reproduction steps.

## Reporting security issues

Please follow the process in [SECURITY.md](./SECURITY.md). Do not open public
issues for security vulnerabilities.
