# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in MaintainerOS,
please report it privately. **Do not open a public issue.**

Email: `TODO_SECURITY_CONTACT_EMAIL`

> The repository owner should replace this placeholder with a real security
> contact before accepting vulnerability reports.

Please include:

- A description of the issue
- Steps to reproduce
- The affected version or commit
- Any potential impact you have identified

We will acknowledge receipt within 3 business days and aim to provide a
remediation plan within 14 days for confirmed issues.

## Scope

This policy covers the MaintainerOS application code in this repository.
Vulnerabilities in upstream dependencies should be reported to their
respective maintainers; we welcome notice so we can update.

## Safe harbor

We will not pursue legal action against researchers who:

- Make a good-faith effort to avoid privacy violations and data destruction
- Only interact with accounts you own or with explicit permission of the owner
- Give us reasonable time to fix the issue before public disclosure
